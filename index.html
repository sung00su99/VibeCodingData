1. 바이브 코딩의 개요

  • AI 코딩은 인공지능을 활용한 모든 코딩 방식을 아우르는 포괄적 개념
  • GitHub Copilot, ChatGPT, Claude, Gemini 등의 AI 도구를 활용해 코드 자동 완성, 버그 탐지, 코드 설명 등을 지원
  • 개발자는 직관과 큰 그림을 담당하고, 구체적인 구현은 AI가 맡는 협업 방식
  • 코드의 작동 원리를 완전히 이해하지 않아도 개발이 가능하지만, '기술을 전혀 몰라도 개발할 수 있다'는 것은 다소 과장에 가까움
  • → 기술을 알아야 이후의 검증을 직접 하거나, 또는 검증 조건을 규정할 수 있음

1. WSL2 (Windows Subsystem for Linux 2) 설치

- 윈도우에서 리눅스 애플리케이션을 구동하게 해주는(정확하게 말하면 Linux system call을 처리해주는) 호환성 레이어로, 리눅스 커널의 기능을 제공

(NT 환경에서 리눅스 OS를 사용할 수 있도록 연결해주는 역할)

1) WINDOWS 기능 설치

(1) 제어판 > 프로그램 > 프로그램 및 기능 > Windows 기능 켜기 또는 끄기

(2) Linux용 윈도우 하위시스템 켜기, 가상 머신 플랫폼 켜기

(3) 프롬프트 환경 실행

cmd.exe

(4) 리눅스 설치

wsl --install Ubuntu-22.04

• 설치 배포 버전 확인

wsl -l -v

• 종료

wsl --terminate Ubuntu-22.04

• 삭제

wsl --unregister Ubuntu-22.04

2. GIT 설정

1) GitHub 계정 생성

https://github.com/

2) 작업 디렉토리 생성 (GitHub에서 Repository 생성)

https://github.com/new

3) 작업 폴더 생성

mkdir ~/work
cd ~/work

4) git 설정

git config --global user.name "사용자이름" git config --global user.email 이메일주소

• 다른 작업자의 변경 사항 Update(pull)을 할 경우 Merge 방식으로 처리

git config --global pull.rebase false

• 적용 사항 확인

git config --global --list

참고 1 > 설정 범위 3단계

범위옵션적용대상저장 위치
시스템 전체 --system PC의 모든 사용자 /etc/gitconfig
현재 사용자 --global 내 계정의 모든 저장소 ~/.gitconfig
현재 저장소 --local 지금 이 프로젝트만 .git/config

5) SSH 키 생성 및 설정

• SSH 키 생성

ssh-keygen

• 생성 키 확인

cat ~/.ssh/id_rsa.pub

• 생성된 공개키를 GitHub에 붙여넣기

Settings → SSH and GPG Keys → New SSH key 클릭 후 붙여넣기

• 연결 확인

ssh -T git@github.com

※ 성공하면 "Hi [username]! You've successfully authenticated…" 메시지 출력

※ SSH 키는 WSL2와 GitHub가 통신할 때 필요한 인증 수단

참고 2 > SSH 방식과 HTTPS 방식의 차이점

구분SSHHTTPS
주소 형태 git@github.com:user/repo.git https://github.com/user/repo.git
인증 수단 SSH 키 (개인키 + 공개키) 아이디 + Personal Access Token
최초 설정 복잡 (키 생성 + GitHub 등록) 간단
매번 인증 불필요 (키가 자동 처리) 토큰 입력 필요 (캐시 설정 전까지)
보안성 높음 보통
방화벽 문제 22번 포트 차단 시 막힘 443번 포트 사용 (거의 안 막힘)
실무 선호도 개발자 환경에서 선호 간단한 작업 / CI 환경에서 사용

※ SSH는 한 번 설정하면 편하고 안전, HTTPS는 설정은 쉽지만 매번 토큰 관리 필요

1. AI Coding Agent 환경 설치

- 코드 작업을 자율적으로 수행하는 AI 도구

• NVM(Node Version Manager) 설치

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

※ NVM: node.js의 버전을 관리해주는 매니저

※ node.js란? 브라우저 밖에서도 JavaScript를 독립적으로 실행할 수 있게 해주는 런타임 환경

※ node.js는 버전이 다를 경우 오류 발생, NVM에서 해당 버전을 관리

• NVM 설치 위치 정보를 터미널에 불러옴

export NVM_DIR="$HOME/.nvm"

• NVM을 현재 터미널에서 실제로 활성화하는 명령어

source "$NVM_DIR/nvm.sh"

• node.js LTS 버전 설치

nvm install --lts

※ LTS(Long Term Support): 장기지원버전으로, 검증된 안정적인 제품을 의미

• node 버전 확인

node -v

• npm 버전 확인

npm -v

1. GitHub 배포

- GitHub에서 제공하는 정적 웹사이트 호스팅 서비스

- 별도의 서버 없이 HTML, CSS, JavaScript로 바로 배포 가능

- 배포를 위해서는 Public Repository 생성 필요 (최초 프로젝트 생성 시 Public이라면 아래 명령어 중 일부는 불필요함)

1) 현재 폴더를 Git 저장소로 초기화

git init   # git clone으로 가져온 경우 불필요 (clone에 이미 포함되어 있음)

2) 배포 파일 선택

git add .

참고 1 > 배포 파일 선택 명령어 비교

명령어의미

4) 브랜치 이름 변경 (master → main)

5) 원격 저장소 연결

git remote add origin git@github.com:username/repo_name.git

6) 배포 (Push)

git push origin main

7) 최신 소스 업데이트 (여러 명이 함께 작업할 때)

1. Supabase 배포

  • Backend-as-a-Service
  • PostgreSQL 기반의 관계형 데이터베이스 제공
  • 인증, DB, 스토리지, 실시간 기능을 통합 제공
  • 서버를 직접 운영하지 않고도 풀스택 개발 가능
  • 2개의 프로젝트까지 무료로 사용 가능

※ 프로젝트에 따라 인증 절차는 다르며, 개발자 상황에 따라 인증 절차 확인 후 처리 필요

1) Supabase 프로젝트 생성

- https://supabase.com 접속 → 회원가입 (GitHub 계정으로 가능)

2) 프로젝트 생성 후 아래 값 별도 저장

이름설명위치
API URL Supabase 프로젝트의 주소 (엔드포인트) Integrations → Data API → API_URL
anon public key 일반 사용자(비로그인/로그인)가 사용하는 키 Settings → API → anon public key
service_role key 관리자 전용 키로 모든 데이터 접근 가능 · 외부 노출 금지 Settings → API → service_role key

3) Table 생성

- SQL Editor 실행 후 Table 생성 (직접 쿼리 실행)

1. Supabase 이메일 인증

  1. Authentication → Providers → Email 활성화
  2. Authentication → URL Configuration → Site URL 에 GitHub Pages URL 입력

    예) https://username.github.io

  3. Authentication → URL Configuration → Redirect URLs 입력

    예) https://username.github.io/repo_name/

※ 이메일 인증 없이 바로 로그인하려면 "Confirm email" 비활성화

2. Google 인증

  1. https://console.cloud.google.com 접속
  2. 프로젝트 생성
  3. APIs & Services → OAuth consent screen → External 선택 → 앱 이름/이메일 입력 후 저장
  4. APIs & Services → Credentials → Create Credentials → OAuth client ID
    • Application type: Web application
    • Authorized redirect URIs 등록 (Supabase API URL 확인)

      ※ https://[프로젝트ID].supabase.co/auth/v1/callback

  5. Supabase에 Google 등록
    • Supabase 대시보드 → Authentication → Providers → Google 활성화
    • Client ID, Client Secret 붙여넣기 → Save

3. GitHub 인증

  1. GitHub → Settings → Developer settings → OAuth Apps → New OAuth App
    • Application name: 자유
    • Homepage URL: https://username.github.io/repo-name/
    • Authorization callback URL: 
      https://[프로젝트ID].supabase.co/auth/v1/callback
  2. 생성 후 Client ID 확인, Generate a new client secret 클릭해서 Client Secret 복사

1. SpecKit 이란?

Spec Kit은 GitHub에서 만든 오픈소스 도구로, Spec-Driven Development(SDD, 사양 중심 개발) 방법론을 실천하게 해주는 CLI 툴킷

1) 설치 및 프로젝트 초기화

- uv 설치

sudo snap install astral-uv --classic

- spec-kit 설치

uv tool install specify-cli --from git+https://github.com/github/spec-kit.git

- 프로그램 초기화

specify init

참조 1 > 개발 워크플로우 진행 순서

명령어설명생성 파일저장 위치
/speckit.specify 프로젝트의 핵심 규칙과 방향성을 설정 spec.md specs/001-feature-name/
/speckit.plan 기술 스택과 아키텍처 선택을 제공하면 AI가 구현 계획 생성 plan.md specs/001-feature-name/
/speckit.task 구현 계획을 실행 가능한 단위 태스크로 분해 tasks.md specs/001-feature-name/
/speckit.implement AI 에이전트가 계획을 실행하여 실제 코드를 생성 - -
